Open with the problem, decision, or useful idea. Give the reader a reason to continue without spending several paragraphs announcing what the article will cover.

<Callout type="tip" title="A useful callout">
  Use callouts for information that deserves emphasis, not for every other paragraph.
</Callout>

## First major section

Keep paragraphs focused. Use lists when the items are genuinely easier to scan as a list.

### A supporting point

Add inline code such as `npm run build`, or a highlighted block:

```ts
export function usefulExample(value: string) {
  return value.trim();
}
```

## Finish with a useful conclusion

Summarise the decision or lesson, then offer a relevant next step. Avoid a generic conclusion that only repeats the introduction.
